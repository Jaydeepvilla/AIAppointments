# Documentation Governance System

## Purpose
This system is the single source of truth for the entire SaaS platform's documentation ecosystem. It dictates how documentation is created, reviewed, managed, and archived. Every AI agent, developer, designer, and QA engineer MUST follow these policies.

## Core Principles
- **Nothing is undocumented**: All features, APIs, and processes must have corresponding documentation.
- **Nothing exists without ownership**: Every document has a defined owner responsible for its accuracy.
- **Nothing becomes obsolete without detection**: Metrics and lifecycles ensure documentation remains fresh.

## Directory Architecture
- `ownership.md`: Department and Agent ownership mapping.
- `documentation-lifecycle.md`: States from Draft to Archive.
- `change-management.md`: Requirements for modifying documentation.
- `quality-standards.md`: Strict rejection criteria.
