# Documentation Metrics

The health of the documentation ecosystem is tracked via the following metrics:

## 1. Coverage
Percentage of platform features/APIs with associated published documentation.
- **Target**: 100%

## 2. Freshness
Percentage of documents updated or verified within the last 90 days.
- **Target**: > 95%

## 3. Broken Links
Count of unresolved internal or external links across the ecosystem.
- **Target**: 0

## 4. Review Status
Percentage of documentation currently in the 'Approved' or 'Published' state versus 'Draft'.
- **Target**: > 80% Published

## 5. Average Age
The average time since the last update across all documents.
- **Target**: < 180 days

## 6. Missing Owners
Count of documents lacking a defined owner.
- **Target**: 0

## 7. Documentation Health Score
A composite index (0-100) factoring in all the above metrics. Scores below 90 trigger a mandatory documentation sprint.
