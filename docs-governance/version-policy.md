# Version Policy

## Documented Versions
- **Latest**: The bleeding edge, typically mapped to the `main` branch. Unreleased features.
- **Stable**: The current production version.
- **LTS (Long Term Support)**: Previous major versions that still receive critical security updates.
- **Archived**: Versions no longer supported.
- **Future**: Planned architecture not yet implemented (strictly cordoned).

## User Visibility
- Users default to viewing **Stable**.
- Users can select **LTS** via a version dropdown.
- **Latest** and **Future** are restricted to internal/developer views unless explicitly flagged.
