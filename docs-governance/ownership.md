# Documentation Ownership

Every documentation category must have an explicit owner responsible for its accuracy and freshness.

## Matrix

| Category | Human Owner | AI Agent Role | Responsibilities |
|---|---|---|---|
| **Product** | VP of Product | Product Manager Agent | Vision, Goals, Feature Overviews, Release Notes. |
| **Engineering** | CTO / Tech Lead | Technical Writer Agent | Architecture, Workflows, Internal Guides. |
| **AI** | Head of AI | AI Architect Agent | AI Behaviors, Prompts, Routing logic. |
| **QA** | QA Lead | QA Engineer Agent | Validation Rules, Test Scenarios, Validation Reports. |
| **Security** | CISO | Technical Writer Agent | Security Boundaries, Permissions, Compliance. |
| **Support** | Head of Support | UX Writer Agent | FAQs, Troubleshooting, User Guides. |
| **Marketing** | CMO | UX Writer Agent | Public Documentation, SEO Optimization. |
| **Operations** | COO | Release Manager Agent | Runbooks, Deployment Pipelines, Changelogs. |
| **Architecture**| Chief Architect | Technical Writer Agent | C4 Diagrams, Data Models, System Context. |

## Enforcement
A document without an assigned owner in its frontmatter is considered **Incomplete** and will be rejected by the Quality Validator.
