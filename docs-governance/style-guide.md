# Style Guide

## Tone and Voice
- Professional, objective, and clear.
- Use Active Voice ("The system processes the data", not "The data is processed by the system").
- Empathetic to the user's context.

## Formatting
- **Headings**: Use sentence case. Do not skip levels (e.g., H1 to H3).
- **Bold**: Use for UI elements the user must interact with.
- **Italics**: Use for introducing new terminology.
- **Code Blocks**: Always specify the language for syntax highlighting.

## Diagramming
- Use Mermaid.js exclusively for architecture and workflow diagrams.
- Avoid complex, unreadable monolithic diagrams; break them down by bounded context.

## UI Design Philosophy
- Borders over shadows. The UI is clean, modern, and enterprise-grade. Documentation should reflect this premium feel in its structural clarity.
